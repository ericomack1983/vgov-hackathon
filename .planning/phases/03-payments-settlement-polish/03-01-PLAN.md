**1. Create `src/lib/settlement-engine.ts`:**

Define settlement state types and reducer:

```typescript
export type USDSettlementStep = 'idle' | 'authorized' | 'processing' | 'settled';
export type USDCSettlementStep = 'idle' | 'submitted' | 'confirmed' | 'settled';

export interface SettlementState {
  method: 'USD' | 'USDC';
  currentStep: USDSettlementStep | USDCSettlementStep;
  progress: number; // 0, 33, 66, 100
  txHash?: string;
  orderId: string;
  startedAt?: string;
}

export type SettlementAction =
  | { type: 'START'; payload: { method: 'USD' | 'USDC'; orderId: string } }
  | { type: 'ADVANCE' }
  | { type: 'RESET' };

export const INITIAL_SETTLEMENT_STATE: SettlementState = {
  method: 'USD',
  currentStep: 'idle',
  progress: 0,
  orderId: '',
};
```

The reducer handles:
- `START`: Sets method, generates txHash for USDC (64 hex chars prefixed with `0x`), sets first step (`authorized` for USD, `submitted` for USDC), progress to 33, records `startedAt` as `new Date().toISOString()`.
- `ADVANCE`: Transitions to next step. USD: authorized->processing (progress 66), processing->settled (progress 100). USDC: submitted->confirmed (progress 66), confirmed->settled (progress 100). If already `settled`, return same state.
- `RESET`: Returns `INITIAL_SETTLEMENT_STATE`.

Export helper constants:
```typescript
export const USD_NODES = [
  { label: 'Government Bank', icon: 'Building' },
  { label: 'Visa Network', icon: 'CreditCard' },
  { label: 'Supplier Bank', icon: 'Building' },
] as const;

export const USDC_NODES = [
  { label: 'Government Wallet', icon: 'Wallet' },
  { label: 'Polygon Network', icon: 'Globe' },
  { label: 'Supplier Wallet', icon: 'Wallet' },
] as const;

export const USD_STEP_DELAY = 2000; // 2s per step, ~6s total
export const USDC_STEP_DELAY = 1500; // 1.5s per step, ~3s total
```

Export `function getStepLabel(step: string): string` that maps each step to a human-readable label: authorized->"Authorized", processing->"Processing (T+2)", settled->"Settled", submitted->"Submitted to Polygon", confirmed->"Confirmed on Chain", idle->"Ready".

**2. Create `src/hooks/useSettlement.ts`:**

`'use client'` directive. Import `useReducer`, `useCallback`, `useEffect`, `useRef` from react, `settlementReducer`, `INITIAL_SETTLEMENT_STATE`, `USD_STEP_DELAY`, `USDC_STEP_DELAY` from settlement-engine, `toast` from `react-hot-toast`, `Transaction` from types.

```typescript
export function useSettlement(onComplete: (tx: Transaction) => void) {
  const [state, dispatch] = useReducer(settlementReducer, INITIAL_SETTLEMENT_STATE);
  const timerRef = useRef<ReturnType<typeof setTimeout> | null>(null);
  const hasCompletedRef = useRef(false);
```

- `start(method, orderId)`: Reset `hasCompletedRef.current = false`, then dispatch START.
- `reset()`: Clear timer if exists, dispatch RESET, reset `hasCompletedRef`.

useEffect on `[state.currentStep, state.method]`:
  - If `idle`, return early.
  - If not `idle` and not `settled`: set timeout with delay (`state.method === 'USD' ? USD_STEP_DELAY : USDC_STEP_DELAY`) to dispatch ADVANCE. Fire `toast.success` with message `${state.method} Payment: ${getStepLabel(state.currentStep)}` and icon `state.method === 'USDC' ? '&#x26D3;' : '&#x1F3E6;'`. Cleanup: clear timeout.
  - If `settled` and `!hasCompletedRef.current`: set `hasCompletedRef.current = true`, fire final toast `${state.method} Payment: Settled`, call `onComplete` with a Transaction object built from state (id: `tx-${Date.now()}`, rfpId/supplierId/supplierName come from closure or pass through, status: 'Settled', settledAt: new Date().toISOString(), etc.).

Note: The `onComplete` callback receives a partial transaction; the page component is responsible for filling in rfpId, supplierId, supplierName, and amount before calling `addTransaction`.

Return `{ state, start, reset, isSettled: state.currentStep === 'settled', isActive: state.currentStep !== 'idle' }`.
