- `IPCPromptRequest`, `IPCRuleSetResponse`, `IPCSuggestedRules`
- `VPCSupplierRegistration`, `VPCSupplierValidation`

**2. `src/payments/VPCService.ts`** — Five sub-services:

AccountManagementService:
- `createAccount(params)` — register card with VPC (required before setting rules)
- `getAccount(accountId)` — retrieve account + contacts
- `updateAccount(accountId, params)` — update card registration
- `deleteAccount(accountId)` — remove account + all rules + notifications

RulesManagementService:
- `setRules(accountId, rules[])` — full replacement of all rules (near real-time)
- `getRules(accountId)` — fetch all rules + SPV/SPP spend usage
- `deleteRules(accountId)` — clear all rules
- `blockAccount(accountId)` — block all transactions, remove rules
- `disableRules(accountId)` — temporarily disable all rules
- `enableRules(accountId)` — re-enable previously disabled rules

ReportingService:
- `getNotificationHistory(accountId, params?)` — email/SMS notification details
- `getTransactionHistory(accountId, params?)` — decline details, merchant info, VPC reason

IntelligentPaymentControls (Gen-AI):
- `getSuggestedRules(prompt)` — Gen-AI returns RuleSetId from natural language prompt
- `setSuggestedRules(ruleSetId, overrides?)` — apply suggested rules with optional modifications

SupplierValidationService:
- `registerSupplier(params)` — submit supplier for validation (Acquirer BIN + CAID)
- `updateSupplier(supplierId, params)` — update validation status
- `retrieveSupplier(acquirerBin, caid)` — get validation status

**3. Rule codes to implement** (from VPC Rule Codes reference):
- `HOT` — Block all transactions
- `SPV` — Spend Velocity (amount + count + period + currency)
- `SPP` — Spend Policy (max single transaction amount + currency)
- `MCC` — Merchant Category Code allow/block list
- `MCG` — Merchant Category Group
- `VPAS` — Exact Match (amount + currency)
- `BHR` — Business Hours (days + hours + timezone)
- `CHN` — Channel (online/POS/ATM/contactless)
- `LOC` — Location (country codes)

**4. Sandbox mode** — all services support sandbox simulation without real credentials

**5. Export from `src/index.ts`** — export VPCService and all VPC types

**6. `examples/vpc.ts`** — full walkthrough: create account → set rules → IPC → report

**7. README** — new section "3 · Visa B2B Payment Controls (VPC)" with onboarding flow, rule codes table, IPC Gen-AI example
