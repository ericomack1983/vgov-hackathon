## Interaction Contracts

### RFP Creation Flow
1. Gov Officer clicks "Create RFP" button on /rfp page
2. Navigates to /rfp/new with CreateRFPForm
3. Form validates: title required, budget > 0, deadline in future, category required
4. On submit: RFP added to ProcurementContext with status "Draft", redirect to /rfp/[id]
5. On /rfp/[id], Gov Officer clicks "Publish RFP" to change status to "Open"

### Bid Submission Flow
1. Supplier sees list of Open RFPs on /bids page
2. Clicks "Submit Bid" on an RFP row, opens BidFormModal
3. Form validates: amount > 0, delivery days > 0
4. On submit: Bid added to RFP in ProcurementContext, modal closes, success toast

### AI Evaluation Flow
1. Gov Officer on /rfp/[id] sees "Run AI Evaluation" button (enabled when >= 2 bids exist)
2. Click triggers: RFP status changes to "Evaluating", 1.5-second simulated loading animation (pulsing indigo bar)
3. Results appear: ranked list with composite scores, radar chart, dimension score grid, "Best Value" badge on winner
4. ExplainabilityPanel renders below with narrative: "Based on [winner name]'s strong [top dimension] score of [value] and competitive pricing at $[amount], they represent the best value. [Runner-up name] scored [X points] lower primarily due to [weakest dimension]."

### Manual Override Flow
1. Gov Officer clicks "Override Recommendation" below the AI results
2. OverrideForm expands with: supplier select dropdown (non-winner suppliers), textarea for justification (required, min 20 characters)
3. On confirm: selected supplier gets winner status, override logged, original AI recommendation preserved for audit
4. Visual: original winner row loses left-border accent, new winner gains it, "Manually Selected" badge replaces "Best Value"

### Page Transitions
All page content uses framer-motion fade+slide-up (opacity 0>1, y 8>0, duration 0.3s, easeOut) consistent with Phase 1 PlaceholderPage pattern.

## Accessibility Notes

- All form inputs must have associated `<label>` elements
- StatusBadge uses semantic color + text (not color alone)
- ScoreBar uses `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, `role="meter"`
- BidFormModal traps focus and closes on Escape key
- Radar chart includes a visually-hidden data table fallback for screen readers
- "Best Value" badge uses `aria-label="AI recommended best value supplier"`
